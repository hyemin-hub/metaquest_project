## 🙏 팀이 GitHub 웹에서 직접 해줘야 할 것 (내가 못 함)

이건 GitHub 페이지 들어가서 사람이 클릭해야 하는 거라 코드로는 못 잡음.

### A. `main` 브랜치 보호 (5분, **hyemin이 해주면 됨**)

1. https://github.com/hyemin-hub/metaquest_project/settings/branches
2. **Add rule** 클릭
3. Branch name pattern: `main`
4. 체크할 옵션:
   - ✅ Require a pull request before merging
   - ✅ Require approvals: **1** 명 이상
   - ✅ Require review from Code Owners ← **이거 켜야 CODEOWNERS 동작함**
   - ✅ Do not allow bypassing the above settings
5. Save changes

### B. CODEOWNERS의 placeholder 실제 아이디로 교체
- `.github/CODEOWNERS` 파일 열어서 `@팀원A-깃허브아이디` 같은 부분을 실제 GitHub 아이디로 바꾸기
- 4명 아이디 확정되면 내가 한번에 PR로 정리 가능

### C. (선택) Issue 라벨 만들기
GitHub → Issues → Labels에서:
- `Must Have` (빨강)
- `Should Have` (주황)
- `Could Have` (노랑)
- `Would Have` (회색)
- `A-physics` / `B-projectile` / `C-managers` / `D-ui` (담당자별)

### D. (선택) MoSCoW 표의 Must/Should 항목을 Issue로 등록
- "성벽 물리 구조" → Issue로 등록, 담당 A 지정, `Must Have` 라벨
- 이런 식으로 MoSCoW 표 9개 항목 다 Issue로 만들면 진행 추적 편함
