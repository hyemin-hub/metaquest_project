## 2. 최초 환경 세팅 (한 번만)

> 처음 프로젝트 합류했을 때만 따라하면 됩니다. 한 번 다 해두면 그 다음부터는 4번 섹션만 보면 됨.

### 2-1. Unity 6 LTS 설치
1. [Unity Hub 다운로드](https://unity.com/download)
2. Unity Hub → **Installs** → **Install Editor** → **Unity 6 LTS** 선택
3. 모듈 체크 (Meta Quest는 Android 기반):
   - ✅ Android Build Support
   - ✅ Android SDK & NDK Tools
   - ✅ OpenJDK

### 2-2. Git 설치
터미널(Mac) 또는 PowerShell(Windows)에서:
```bash
git --version
```
- **Mac:** 위 명령 치면 자동 설치 팝업
- **Windows:** [git-scm.com](https://git-scm.com/download/win) 에서 다운로드

### 2-3. Git LFS 설치 ⚠️ **필수**

LFS 없으면 fbx, psd, wav 같은 큰 파일이 **깨진 텍스트**로 받아져요.

**Mac:**
```bash
# Homebrew 없으면 먼저 설치
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install git-lfs
```

**Windows:** [git-lfs.com](https://git-lfs.com) 에서 다운로드

**공통 — 설치 후 한 번만:**
```bash
git lfs install
```
`Git LFS initialized.` 메시지 뜨면 OK.

### 2-4. GitHub 권한 받기
- hyemin한테 본인 GitHub 아이디 알려주고 collaborator로 추가 요청
- 이메일로 초대 메일 오면 **Accept**

### 2-5. 레포 클론
```bash
git clone https://github.com/hyemin-hub/metaquest_project.git
cd metaquest_project
```

### 2-6. Unity Hub에서 열기
1. Unity Hub → **Projects** → **Add** → 클론한 폴더 선택
2. 처음 열 때 패키지 설치하느라 5~10분 걸림. 기다리기.

### 2-7. Unity 설정 확인
`Edit → Project Settings → Editor`에서:

| 항목 | 값 |
|------|------|
| Asset Serialization → Mode | **Force Text** |
| Version Control → Mode | **Visible Meta Files** |

### 2-8. UnityYAMLMerge (Smart Merge) 등록 ⚠️ **필수**

씬·프리팹 머지 충돌 났을 때 Unity가 자동으로 합쳐주게 하는 설정. 안 해두면 충돌 났을 때 텍스트 그대로 떠서 손으로 풀어야 함.

**Mac:**
```bash
git config --global merge.unityyamlmerge.name "Unity SmartMerge"
git config --global merge.unityyamlmerge.driver '/Applications/Unity/Hub/Editor/<Unity버전>/Unity.app/Contents/Tools/UnityYAMLMerge merge -p %O %B %A %A'
git config --global merge.unityyamlmerge.recursive binary
```

**Windows (PowerShell):**
```powershell
git config --global merge.unityyamlmerge.name "Unity SmartMerge"
git config --global merge.unityyamlmerge.driver '"C:/Program Files/Unity/Hub/Editor/<Unity버전>/Editor/Data/Tools/UnityYAMLMerge.exe" merge -p %O %B %A %A'
git config --global merge.unityyamlmerge.recursive binary
```

> `<Unity버전>` 자리는 본인이 깐 Unity 6 LTS 정확한 버전 (예: `6000.0.30f1`). Unity Hub → Installs 탭에서 확인.

### 2-9. Meta XR SDK 설치 (B 담당 필수 / 나머지 권장)
1. Unity Editor → `Window → Package Manager`
2. 좌상단 `+` → `Add package by name...`
3. `com.meta.xr.sdk.all` → Add

> 기본 Unity XR Interaction Toolkit / Oculus XR Plugin은 이미 `Packages/manifest.json`에 들어있어서 자동 설치돼요. Meta XR SDK는 라이선스 동의 때문에 각자 따로 설치 필요.

## 3. 팀원 분담 & 폴더/씬 구조

### 3-1. 팀원별 담당

| 팀원 | 담당 영역 | 만드는 파일 | 작업 씬 |
|------|------|------|------|
| **A** | 물리 / 성벽 시스템 | `WallBlock.cs`, 성벽 Prefab | `Test_A_Scene` |
| **B** | VR 인터랙션 / 투사체 | `Projectile.cs`, `ExplosiveProjectile.cs`, 공 Prefab | `Test_B_Scene` |
| **C** | 게임 매니저 / 가챠 | `GameManager.cs`, `GachaManager.cs` | `Test_C_Scene` |
| **D** | UI / 최종 통합 | `UIManager.cs`, MainScene 관리 | `MainScene` |
| hyemin | 협업 인프라 | `.github/`, `.gitignore`, `.gitattributes`, README | (관리만) |

### 3-2. 씬 분리 (1인 1씬)

| 씬 | 담당 | 용도 |
|------|------|------|
| `MainScene` | **D 전용** | 최종 통합. **A·B·C 절대 수정 금지** |
| `Test_A_Scene` | A | 성벽 물리 테스트 |
| `Test_B_Scene` | B | VR 투사체 / 인터랙션 테스트 |
| `Test_C_Scene` | C | 매니저 / 가챠 로직 테스트 |

> ⚠️ MainScene은 `CODEOWNERS`로 보호되어 있어서 D 승인 없이는 머지 안 됨. 실수로라도 수정하지 말기.

### 3-3. 폴더 구조

```
Assets/
├── Scenes/         (MainScene + Test_A/B/C_Scene)
├── Scripts/
│   ├── Physics/    (WallBlock.cs 등 — A)
│   ├── Projectiles/(Projectile.cs 등 — B)
│   ├── Managers/   (GameManager.cs, GachaManager.cs — C)
│   └── UI/         (UIManager.cs — D)
├── Prefabs/
│   ├── Walls/      (성벽 블록 — A)
│   └── Projectiles/(공/폭탄 등 — B)
├── Materials/Physics/  (나무/돌 Physic Material — A)
├── Audio/{SFX, BGM}/   (효과음·배경음 — 3번)
└── Models/             (Meshy로 뽑은 3D 모델)
```

### 3-4. 통합 흐름

```
[A] Test_A_Scene에서 성벽 프리팹 완성 → push (Prefab + Script만)
[B] Test_B_Scene에서 투사체 프리팹 완성 → push (Prefab + Script만)
[C] Test_C_Scene에서 매니저 로직 완성 → push (Script만)
              ↓
[D] 모든 PR pull → MainScene 열고 프리팹·매니저 배치 → Quest 빌드 테스트
```

## 4. 매일 쓰는 협업 규칙

### 4-1. 협업 5계명 ⚠️

1. **같은 Scene / Prefab 동시 수정 금지** — 작업 전에 슬랙/카톡으로 "나 ○○ 수정 시작" 공지
2. **자주 pull, 자주 commit** — 하루치 작업을 한꺼번에 올리면 충돌 폭발
3. **파일 이동은 반드시 Unity Editor 안에서** — Finder/탐색기로 옮기면 `.meta` 꼬여서 참조 다 깨짐
4. **`.meta` 파일은 항상 같이 커밋** — 스크립트/프리팹 추가했으면 `.meta`도 반드시
5. **큰 바이너리는 신경 안 써도 됨** — `.gitattributes`가 자동으로 LFS로 보내줌

### 4-2. 작업 흐름 (매번 반복)

```bash
# 1. 작업 시작 전: 최신 받기
git checkout main
git pull origin main

# 2. 새 브랜치 만들기 (본인 영역 prefix)
git checkout -b feature/A-wall-hp-system
#                     ↑   ↑
#                  본인 알파벳  작업 내용

# 3. Unity에서 작업

# 4. 작업 끝나면 커밋
git add .
git commit -m "feat: 성벽 블록 HP 시스템 추가"

# 5. 원격에 올리기
git push -u origin feature/A-wall-hp-system

# 6. GitHub에서 PR 생성 → 팀원 리뷰 → 머지
```

### 4-3. 브랜치 네이밍

| 형식 | 예시 |
|------|------|
| `feature/A-작업명` | `feature/A-wall-physics` |
| `feature/B-작업명` | `feature/B-throw-mechanic` |
| `feature/C-작업명` | `feature/C-gacha-random` |
| `feature/D-작업명` | `feature/D-scoreboard-ui` |
| `fix/X-작업명` (버그) | `fix/B-collision-bug` |

### 4-4. 커밋 메시지 컨벤션

| 접두어 | 용도 | 예시 |
|------|------|------|
| `feat:` | 새 기능 | `feat: 활 발사 시스템 추가` |
| `fix:` | 버그 수정 | `fix: 화살 충돌 판정 오류 수정` |
| `chore:` | 설정/잡일 | `chore: gitignore 업데이트` |
| `docs:` | 문서 | `docs: README 업데이트` |
| `refactor:` | 코드 정리 | `refactor: 발사체 코드 구조 변경` |

## 5. PR 올리는 법

### 5-1. PR 생성
1. `git push` 한 뒤 터미널에 뜨는 GitHub URL 클릭 (또는 GitHub 페이지에서 직접 New PR)
2. base는 `main`, compare는 본인 `feature/...` 브랜치
3. 자동으로 뜨는 **PR 템플릿** 작성:
   - 작업 내용
   - 관련 이슈 (있으면)
   - 작업 영역 체크
   - 체크리스트 (MainScene 안 건드림, .meta 챙김 등)

### 5-2. PR 체크리스트 (올리기 전 확인) ✅
- [ ] 본인 Test_X_Scene에서 동작 확인했는가
- [ ] **MainScene 수정 안 했는가** (D 제외)
- [ ] `.meta` 파일 같이 커밋했는가
- [ ] 큰 바이너리 파일이 LFS로 들어갔는가 (`git lfs ls-files`로 확인)
- [ ] Finder/탐색기가 아니라 Unity Editor에서 파일 이동했는가
- [ ] 커밋 메시지가 컨벤션 따랐는가

### 5-3. 리뷰 → 머지
- 같은 영역 담당자한테 리뷰 요청 (`Reviewers` 패널에서)
- D는 모든 PR을 한 번씩 훑어보기 (통합 담당이라)
- Approve 받으면 **Squash and merge** 권장 (커밋 히스토리 깔끔)

## 6. 자주 발생하는 문제 (Troubleshooting)

### ❓ `git push`가 `permission denied`로 거부됨
→ Collaborator 권한 못 받은 상태. hyemin한테 요청.

### ❓ 클론했는데 `.fbx`, `.psd` 파일이 깨진 텍스트로 보임
→ Git LFS 설치 안 됨. **2-3 다시 확인** 후:
```bash
git lfs install
git lfs pull
```

### ❓ Scene 또는 Prefab 머지 충돌 떴을 때
1. 패닉 금지
2. 슬랙/카톡에 충돌 공지
3. 가능하면 한쪽 버전을 살리고 나머지 작업은 다시 수동 적용
4. 정 안 되면 Unity Smart Merge 활용 (2-8에서 등록한 거)

### ❓ `.meta` 파일이 꼬였을 때
- 파일 이동을 Finder/탐색기로 한 경우 자주 발생
- **무조건 Unity Editor 안에서만 파일 이동**
- 이미 꼬였으면 GUID가 바뀌어서 참조가 깨질 수 있음. 슬랙/카톡에 공유하고 같이 복구.

### ❓ Unity가 갑자기 느려지거나 이상해짐
1. Unity 종료
2. 프로젝트 폴더의 `Library/` 폴더 통째로 삭제 (Unity가 다시 만들어줌)
3. Unity 다시 열기 (재빌드에 시간 좀 걸림)

### ❓ `git add` 할 때 `index.lock` 에러
```
fatal: Unable to create '.git/index.lock': File exists.
```
→ 이전 git 프로세스가 비정상 종료된 거. lock 파일 지우면 됨:
```bash
rm -f .git/index.lock
```

### ❓ 메인에 안 올라간 것 같음 (`Everything up-to-date`)
→ commit이 안 된 상태에서 push한 거. `git status`로 변경사항 확인하고 `git add . && git commit` 다시.

### ❓ Quest 빌드 시 XR 에러
→ Meta XR SDK 설치 안 함. **2-9** 다시 확인. 또는 `Edit → Project Settings → XR Plug-in Management`에서 **Oculus** 체크 확인.
